# ADR 0002: Модульний моноліт з обмеженими контекстами Core та Analytics

- Status: Accepted
- Date: 2026-04-20
- Scope: Перебудова проєкту в межах лабораторної 6

## Контекст
Еволюція проєкту

## Рішення
1. Розділити систему на обмежені контексти в межах одного розгортуваного проєкту:
   - Core: users/auth, movies, payments, subscription
   - Analytics: read-only проєкції та метрики
2. Визначити публічні контракти через API-пакети модулів:
   - `payments.api.event.*`
   - `subscription.api.event.*`
   - `analytics.api.*`
3. Зберігати внутрішню реалізацію модулів за межами публічного контракту в пакетах `internal`.
4. Використати ACL в Analytics для мапінгу публічних подій Core у внутрішні моделі Analytics.
5. Використати асинхронну подієво-орієнтовану міжмодульну комунікацію для eventual consistency.

## Наслідки
- Позитивні:
  - Зменшення зв'язаності між обмеженими контекстами.
  - Безпечніша еволюція внутрішніх моделей.
  - Модуль Analytics можна виділити в окремий сервіс із мінімальними змінами.
- Негативні:
  - Більше класів і мапінгу (ACL + API-контракти).
  - Eventual consistency між модулями потребує ідемпотентності та моніторингу.

## Модель консистентності
- Strong consistency: у межах транзакції одного модуля (payment -> subscription).
- Eventual consistency: між Core і Analytics через події.
